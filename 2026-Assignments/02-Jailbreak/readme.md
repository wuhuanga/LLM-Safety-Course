# 方向02：大模型越狱攻防（LLM Jailbreak Attack & Defense）

## 题目描述

构建一个面向大语言模型安全性的越狱攻防（Jailbreak Attack & Defense）研究项目。选择一类目标模型（如开源 Qwen-2.5、LLaMA-3.1、Gemma-2 或经授权可调用的 API 模型），围绕"如何诱导模型违反安全策略"以及"如何让模型在面对此类诱导时仍然安全可控"两条主线开展研究。学生可任选以下三条赛道之一：

- **赛道 A（红队 / 攻击）**：复现并改进至少两类越狱攻击方法（如 GCG、AutoDAN、PAIR、Crescendo、多语言/编码绕过、角色扮演 DAN 系列等），分析攻击的可迁移性与失效边界。
- **赛道 B（蓝队 / 防御）**：设计并实现可部署的越狱防御方案（如输入/输出分类器、Perplexity 检测、SmoothLLM、系统 Prompt 加固、基于 SFT/DPO 的安全对齐微调等），在真实攻击集上评估防御效果。
- **赛道 C（红蓝对抗）**：同时实现攻击与防御，构建闭环对抗演化实验，量化攻防双方的相互影响。

## 题目要求

- **数据准备**：使用公开越狱评测基准（如 HarmBench、AdvBench、JailbreakBench、StrongREJECT），并搭配良性查询集（如 XSTest、AlpacaEval）评估对正常请求的过度拒答率（Over-refusal）。如需自建样本，请控制在标准基准已覆盖的有害类别范围内，**禁止编写真实可用于实施违法行为的具体指引**。
- **方法实现**：至少完整实现 2 个攻击方法或 2 个防御方法（赛道 C 至少各 1 个）；要求源代码可复现，附上超参与随机种子。
- **核心评估**：报告必须包含 ASR（Attack Success Rate）、拒答率、对良性查询的误拒率（FPR）、推理延迟等量化指标，并使用 **LLM-as-a-Judge** 或人工标注双重校验。
- **迭代优化**：可通过更新攻击模板、检测特征、防御 Prompt 或微调数据持续提升效果，记录每一轮迭代的指标曲线。
- **部署**：使用 Gradio/Streamlit/FastAPI 构建 Web Demo，攻击演示需明确标注"研究用途"，并对输出做长度/类别截断；防御演示需展示拦截过程与可解释依据。

## 安全与伦理约束（必读）

- **仅在开源模型或经授权的 API 上进行测试**，严禁针对未经授权的线上商用模型实施真实攻击。
- 报告与代码中**不得包含可直接复现的危险任务输出全文**，涉及 CBRN（化生放核）、未成年人相关内容等高危类别时，仅以编号或截断片段呈现。
- 项目仓库需包含 `RESPONSIBLE_USE.md`，声明研究目的、数据来源与免责条款。
- 若发现针对主流闭源模型的新型可复现漏洞，须遵循**负责任披露（Responsible Disclosure）**流程，先与助教/老师沟通后再决定是否公开。
