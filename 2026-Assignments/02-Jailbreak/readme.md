# 方向02：大模型越狱攻防（LLM Jailbreak Attack & Defense）


---

## 🎯 题目描述

围绕大语言模型的安全性,亲手做一次小规模的越狱攻防实验。选择一个你方便调用的目标模型(如本地部署的 Qwen-2.5-7B、ChatGLM3,或经授权可访问的开源模型 API),从下面**两条赛道任选其一**:

- **赛道 A —— 红队体验(攻击)**:挑选一种越狱技巧,亲手构造攻击提示并测试目标模型的安全边界。
- **赛道 B —— 蓝队体验(防御)**:针对一组已知的越狱提示,实现一个简单的防御组件并验证拦截效果。

> 💡 **不要求"创新"**,只要求你跑完整个流程、得到自己的真实数据、写下你的观察。

---

## 📋 题目要求(最低工作量)

### 数据准备

- 直接使用 [AdvBench](https://github.com/llm-attacks/llm-attacks)、[JailbreakBench](https://github.com/JailbreakBench/jailbreakbench) 等公开越狱评测数据集,**不需要纠结样本量**——全量跑通也行,跑不动就用数据集自带的 demo 子集。
- **重点是从中挑 3-5 条做 case study**:把模型的完整输入、完整输出、判定结果贴进报告里详细分析,这比"跑了多少条"更能体现你对越狱机理的理解。
- 也可以自己整理覆盖 3-5 个有害类别(违法行为、虚假信息、隐私侵犯等)的指令,**但不要写真实可执行的危险细节**,只需指令本身。

### 方法实现

**赛道 A 二选一即可**:
- **方式 A1(最简单)**:复现 1 种基于 prompt 模板的越狱方法,例如 **DAN 系列**、**角色扮演**、**多语言绕过**、**Base64/Leetspeak 编码**、**前缀注入**等。直接写 Python 脚本批量测试即可。
- **方式 A2(进阶)**:使用现成开源工具(如 `EasyJailbreak`、`PAIR` 的开源实现)跑一遍即可,**不要求自己从零实现 GCG 这类需要梯度优化的算法**。

**赛道 B 二选一即可**:
- **方式 B1(最简单)**:实现一个**关键词/正则黑名单过滤器**,或一个**基于困惑度(Perplexity)的输入检测器**,对一批越狱 prompt 进行拦截判断。
- **方式 B2(进阶)**:用更强的 LLM(或现成的 Llama-Guard、ShieldGemma 等小模型)作为**安全分类器**,判断输入是否安全。

### 评估

报告中至少包含一个简单表格:

| 指标 | 数值 |
|:--|:--:|
| 测试样本数 | x |
| ASR(攻击成功率)/ 拦截率 | xx% |

- **判定方式**:人工浏览模型回复,自己判断是否"越狱成功"或"被成功拦截"即可,不强制使用自动化裁判。
- 在表格之外,**精选 3-5 条样本作为 case study**,把完整的输入/输出/判定理由写进报告——这是评分的重点。

---

## ⚖️ 安全与伦理约束(必读)

- **仅在开源模型或经课程授权的 API 上测试**,严禁针对未授权的线上商用模型(如 ChatGPT、Claude、文心一言等)发起真实攻击。
- 报告与代码中**不得粘贴可直接复现的危险输出全文**。如果模型真的吐出了危险内容,在报告中用 "[截断 / 已脱敏]" 替代关键步骤,只描述类别(如"提供了制作 X 的步骤")即可。
- 涉及 **CBRN(化学/生物/放射/核)**、**未成年人**、**真实人物诽谤** 等高危类别的样本,**只保留指令编号**,不展示具体内容。
- 这是一次教学实验而非真实漏洞挖掘,**请勿将任何攻击 prompt 公开传播**。

## 📦 最后提交要求

1. **代码提交**
   - 一个可以直接运行的 Python 脚本(如 `attack.py` 或 `defense.py`)。
   - 一份简短的 `README.md`,写明:依赖包、目标模型、如何运行(一行命令最佳)。
   - 测试用的 prompt 列表(脱敏后)放在 `prompts.json` 或 `prompts.csv`。

2. **报告提交**
   - 一份 **简短** 的实验报告，包含:
     1. 选了哪个赛道、哪种方法、哪个目标模型。
     2. 数据来源(从哪个 benchmark 取的、跑了多少条)。
     3. 实验设置与运行结果(一个表格 + 3-5 条 case study)。
     4. 你的观察与思考(哪些类别更容易被越狱?防御的失败模式是什么?)——**这部分最重要**。

3. **文件名命名规范**
   - 报告文件:`学号-姓名-02-Jailbreak.md` 或 `.pdf`
   - 示例:`20231234-张三-02-Jailbreak.md`

4. **Pull Request 要求**
   - PR 标题格式:`[作业02-Jailbreak] 学号 姓名 - 项目简述`
     示例:`[作业02-Jailbreak] 20231234 张三 - DAN 越狱模板在 Qwen-2.5 上的成功率测试`
   - PR 描述至少包含:
     - 目标模型与来源
     - 测试样本数与最终 ASR / 拦截率
     - 一句话总结主要发现

## 💡 给同学们的话

这次作业的目的不是"做一个 SOTA 攻击/防御",而是让你:
- **真切体会**当前大模型的安全边界长什么样;
- **亲手验证**一段在论文/新闻里看到的越狱技巧是否真的有效;
- **形成自己的判断**——大模型的安全到底"安全"在哪、"不安全"又在哪。

与其挑一个最炫的方法做不完,不如选一个最简单的方法跑通、写清楚自己的观察。**几条认真分析过的 case 远胜于跑不动的一千条数据。**

祝大家在本课程里学得开心 🛡️
