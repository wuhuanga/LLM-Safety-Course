# 方向02：大模型越狱攻防（LLM Jailbreak Attack & Defense）

## 题目描述
针对开源大语言模型开展越狱攻防研究，要求覆盖**黑盒自动化优化**与**白盒梯度优化**两类不同范式的攻击方法，并实现一种防御机制对其进行抵御与评估。

## 题目要求
- **数据准备**：使用公开越狱评测数据集（如 AdvBench、HarmBench、JailbreakBench），覆盖 ≥ 4 个有害类别；搭配 XSTest 等良性集用于评估过度拒答。
- **模型选择**：选用白盒可访问的开源模型（如 Vicuna-7B、LLaMA-3.1-8B、Qwen2.5-7B 等），显存有限可使用 1-2B 小尺寸版本。
- **核心功能**：实现 ≥ 1 种**黑盒自动化攻击**（如 PAIR 基于攻击者 LLM 的迭代改写、AutoDAN 基于遗传算法的对抗 prompt 进化）+ ≥ 1 种**白盒梯度攻击**（如 GCG 对抗性后缀优化），并实现 1 种防御机制（如 Perplexity 检测、SmoothLLM 输入扰动、Llama-Guard 安全分类）。
- **迭代优化**：可探索攻击的跨模型迁移性、后缀长度/迭代步数对 ASR 的影响，或防御阈值与误拒率的权衡。
- **部署**：构建命令行或 Web Demo，支持输入 prompt 实时展示模型回复、防御判定与拦截依据。
- **评估**：核心指标包括 ASR（攻击成功率）、FPR（良性查询误拒率）、Utility Cost（防御推理延迟开销）；ASR 判定使用 Llama-Guard-3 自动判定 + 人工抽样复核 ≥ 10%。报告需包含 ≥ 3 条 case study（黑盒攻击成功 / 白盒攻击成功 / 被防御拦截 各 ≥ 1 例）。

## 安全与伦理约束
- **仅在开源模型或经课程授权的 API 上测试**，严禁针对未授权的商用闭源模型发起系统性攻击。
- 报告与代码中**不得粘贴可直接复现的危险输出全文**，关键步骤以 `[已脱敏]` 替代；高危类别样本仅保留指令编号。
- GCG 等方法生成的对抗性 prompt 仅用于教学评估，**不得在公开渠道传播**。
