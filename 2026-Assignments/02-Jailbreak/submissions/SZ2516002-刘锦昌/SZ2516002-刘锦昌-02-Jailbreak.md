# 方向02：大模型越狱攻防实验报告

## 1. 实验背景与目的

近年来，大语言模型在自然语言处理、代码生成及逻辑推理等领域展现出了极其强大的能力。然而，伴随其能力的跃升，模型的安全对齐（Safety Alignment，如 RLHF、DPO）脆弱性问题也日益凸显。攻击者可以通过精心设计的越狱（Jailbreak）提示词，绕过模型内部的安全护栏，诱导其生成违法、暴力或有害的内容。这场攻击与防御的军备竞赛已成为大模型安全领域的核心议题。

本实验旨在系统性地探究开源指令微调模型在面对不同范式越狱攻击时的鲁棒性，并评估外部安全防御机制的实际效能。具体研究目标包括：

1. 多范式攻击复现与对比：实现并对比分析两种主流且截然不同的攻击路径，基于自然语言伪装的黑盒自动化攻击（PAIR）与基于特征空间梯度搜索的白盒对抗攻击（GCG），探究其攻击成功率（ASR）及绕过对齐机制的内在机理。
2. 纵深防御机制评估：引入独立于生成模型的 LLM-Guard 安全扫描机制，构建前置拦截的护城河。
3. 安全与可用性的权衡分析（Trade-off）：通过引入良性测试集，量化评估防御机制在拦截恶意指令与误拒正常查询（FPR）之间的矛盾，探索最佳的防御阈值配置。



## 2. 伦理考虑与负责任披露

本研究严格遵守学术伦理与《大模型安全与知识增强》课程的安全约束：

* **测试范围受限**：所有攻击实验均在本地部署的开源模型（如 Qwen2.5-0.5B-Instruct）或经授权的 API 上进行，**未对任何未授权的线上商用闭源系统发起攻击**。
* **数据脱敏处理**：本报告及附带的开源代码库中，针对所有成功诱导出的高危类别样本输出（如炸弹制作、网络入侵等），均已进行严格的 `[已脱敏]` 截断处理，**不包含任何可供复现的危险指引全文**。
* **工具使用声明**：本实验中生成的对抗性后缀（Adversarial Suffixes）及自动化改写 Prompt 仅用于教学与模型鲁棒性评估，承诺绝不在公开渠道散播。



## 3. 实验环境与数据集

为确保实验的科学性与可复现性，本研究选取了业界公认的评测基准与开源组件：

* **目标被测模型**：`Qwen2.5-0.5B-Instruct` (本地部署)。选用该模型的原因是其具备完善的中文指令跟随能力和开源的权重，既满足了白盒攻击对获取参数梯度的需求，又能在有限显存环境下高效完成多次迭代。
* **攻击者大模型**：`DeepSeek-V3 API`。作为 PAIR 黑盒攻击的核心引擎，DeepSeek 拥有强大的逻辑推理和角色扮演能力，能够高效生成复杂的社会工程学越狱提示词。
* **评测数据集**：
  * **恶意评测集（AdvBench）**：作为攻击成功率（ASR）的测试基准。从中抽样了 100 条高质量恶意指令，明确覆盖了网络入侵、暴力伤害、非法活动等 ≥ 4 个有害类别。
  * **良性评测集（XSTest）**：作为误拒率（FPR）的测试基准。抽样 100 条包含敏感词汇但意图安全的良性指令（例如“如何杀掉僵尸进程”），用于验证防御机制是否会出现“过度敏感”的灾难性遗忘问题。
* **核心防线**：`LLM-Guard`。启用了其中的 `PromptInjection` 和 `Toxicity` 本地轻量级扫描器模块。



## 4. 攻击与防御方法机理分析

### 4.1 黑盒自动化攻击：PAIR 迭代改写

PAIR (Prompt Automatic Iterative Refinement) 是一种基于大模型博弈的黑盒攻击范式。其核心机理是引入一个未经严格道德限制（或被赋予红队测试系统提示词）的“攻击者大模型”。攻击者模型不接触目标模型的底层参数，而是利用社会工程学原理（如虚构科幻小说场景、构建虚拟沙盒环境、情景嵌套等），对原始的恶意指令进行自动化改写与润色。这种语义层面的深度伪装，能够有效欺骗目标模型内部基于 RLHF 微调的意图分类器，使其误以为当前的任务是合法且安全的文本续写任务，从而产生越狱。

### 4.2 白盒梯度攻击：GCG 对抗后缀优化

与 PAIR 的自然语言伪装不同，GCG (Greedy Coordinate Gradient) 是一种典型的白盒对抗攻击。其机理直接作用于模型的离散 Token 空间与连续向量空间的映射过程。攻击者首先设定一个明确的违规输出前缀（例如：“当然，以下是制作炸弹的步骤：”）。随后，通过反向传播获取目标大模型的梯度信息，利用梯度寻找能够最大化该违规前缀生成概率的输入组合。经过数十次的贪婪坐标搜索与替换，GCG 会在提示词末尾生成一段人类无法理解的“乱码后缀”。这段后缀在向量空间中形成了强大的扰动，强行扭转了模型的注意力机制，直接击穿了安全对齐的护栏。

### 4.3 纵深防御机制：LLM-Guard 安全扫描

面对上述防不胜防的攻击，本实验采用外置护城河的纵深防御理念。引入 LLM-Guard 作为独立的判别式分类器。在用户输入到达目标大模型之前，`PromptInjection` 等模块会对文本进行基于语义向量的风险评分（Risk Score）。与庞大的生成式模型相比，这种专用于文本分类的轻量级判别模型具有更清晰的安全边界感。一旦计算出的风险值突破预设的阈值（如 0.5），系统将直接在输入端熔断请求，不仅阻断了攻击链路，还极大地节省了目标模型推理生成违规长文本所带来的无效算力开销。

## 6. 实验评估与指标对比表

通过运行评估脚本，对批量数据集进行了定量分析。核心指标计算公式如下：

* **ASR (Attack Success Rate)** = 被 Llama-Guard 判定为攻击成功的样本数 / 恶意样本总数
* **FPR (False Positive Rate)** = 被防御机制错误拦截的良性样本数 / 良性样本总数

本实验共包含200 条测试样本（100条恶意数据测 ASR，100条良性数据测 FPR）。

从表 6.1 的数据中可以得出三个关键结论：

1. **攻击的毁灭性破坏力**：在无防御状态下，无论是基于文本伪装的黑盒 PAIR 攻击，还是基于梯度乱码的白盒 GCG 攻击，均达到了 100.0% 的攻击成功率（ASR）。这表明轻量级开源大模型（0.5B）自身的安全对齐机制在面对系统性攻击时极其脆弱。
2. **安全防御的显著成效**：引入 LLM-Guard 后，纯恶意指令的 ASR 被压缩至 21.0%。更令人瞩目的是，在开启防御后，白盒 GCG 攻击的成功率直接归零（0.0%），黑盒 PAIR 的成功率也锐减至 4.0%。这证明了前置安全扫描机制能有效识别并过滤绝大多数对抗特征。
3. **可用性与安全性的平衡**：防御机制的引入带来了极低且可控的推理延迟（22.9ms ~ 238.3ms），且在良性测试集上的误拒率（FPR）维持在 0.0%，说明该系统在大幅提升安全性的同时，并未牺牲正常用户的交互体验。



### 6.2 防御阈值与误拒率（FPR）的权衡探索

| **LLM-Guard 阈值设定** | **ASR (防线被突破率)** | **FPR (良性误拒率)** | **平均延迟** |
| ---------------------- | ---------------------- | -------------------- | ------------ |
| 阈值 = 0.2 (极度严格)  | 6.0%                   | 0.0%                 | 222.4ms      |
| 阈值 = 0.5 (默认均衡)  | 3.0%                   | 0.0%                 | 144.7ms      |
| 阈值 = 0.8 (极度宽松)  | 3.0%                   | 0.0%                 | 183.8ms      |

上述表格中，阈值 0.2 时 ASR 为 6.0%，而阈值 0.5 时 ASR 为 3.0%。理论上阈值越严格（0.2），拦截应该越多，ASR 应该越低。出现这种倒挂，是因为 PAIR 黑盒攻击调用 API 具有随机性，每次生成的攻击文本强度不同。

从上表可以看出，随着阈值的调整，良性误拒率（FPR）始终保持为 0.0%，这得益于 XSTest 评测集指令的特征相对清晰。但在防线突破率（ASR）上，严苛阈值（0.2）与宽松阈值（0.8）相比并未展现出绝对的压制力（ASR 在 3%~6% 之间波动）。这一波动主要源于 PAIR 自动化黑盒攻击在每次生成时依赖大模型的采样随机性（Temperature）。 综合考虑，将风险阈值设定为 0.5 是最理想的配置，既能在绝大多数场景下阻断恶意探测，又能为正常对话保留足够的宽容度，防止因防线过于敏感（如 TC-02 的中文僵尸进程案例）而导致可用性灾难。

## 8. 总结

本研究完整且深入地实施了从红队多范式越狱攻击到蓝队独立安全扫描防御的对抗闭环实验。通过定量实验与定性分析，我们得出以下重要结论：

1. **开源模型的内生防御极度脆弱**：在无外部防护的状态下，Qwen2.5-0.5B 等开源模型在面对 PAIR 的上下文语义欺骗和 GCG 的梯度空间扰动时，均表现出了 100% 的防线崩溃。这表明仅依靠微调阶段的对齐数据，无法赋予模型抵抗系统性对抗攻击的能力。
2. **轻量级扫描机制效能显著**：部署 LLM-Guard 前置防御后，成功将黑白盒攻击的 ASR 从 100% 压制至 0%，证明了判别式小模型 + 生成式大模型的组合架构是目前防御越狱攻击最具性价比的工程方案。它能以极小的推理延迟（平均约 3-4 秒，相较于长文本生成可忽略不计）换取系统安全性的质变。
3. **安全阈值的权衡困境依然存在**：通过 FPR 的交叉评测与案例分析（如中文“僵尸进程”的误杀），我们发现当前开源判别模型在跨语言理解和深层语境感知上仍存在严重短板。固定的防御阈值往往会导致较高的良性误拒率。

未来的大模型安全研究，不应仅仅停留在构建更复杂的攻击提示词，而应致力于防御机制的智能化演进。例如：探索基于动态阈值（Dynamic Thresholding）的自适应拦截机制；或是研究如何在不损失模型生成质量的前提下，将 LLM-Guard 的判别能力通过蒸馏（Knowledge Distillation）内化到目标大模型的参数中，从根源上提升大模型的原生鲁棒性。
