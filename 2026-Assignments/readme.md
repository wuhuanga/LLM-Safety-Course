# 🛡️ 《大模型安全与知识增强》课程实验仓库

> 本仓库是《大模型安全与知识增强》课程的实验项目工作区。整个学期围绕 **一个综合性大项目** 展开——你将从下面三个方向中**任选其一**，全程迭代地构建一个具有真实研究价值的系统，并在学期结束时以 Pull Request 的形式提交最终成果。

------

## 🎯 课程实验目标

- 系统理解大模型在 **能力扩展（Agent）**、**安全攻防（Jailbreak）**、**知识维护（Knowledge Editing）** 三个维度的核心技术原理。
- 完整经历"选题 → 文献调研 → 方法实现 → 实验评测 → 报告撰写 → 公开提交"的研究闭环。
- 学会用 Git / GitHub 进行真实的代码协作与版本管理，积累一份可写入简历的开源协作记录。
- 培养工程化、可复现的实验习惯，并在过程中持续优化指标。

## 📥 提交规范

### Step 1：Fork & Clone

将本仓库 Fork 到你自己的 GitHub 账号下，然后克隆到本地。如果不熟悉 Git 操作，请先阅读 [`tutorial.md`](https://claude.ai/chat/tutorial.md)。

### Step 2：选择方向与开发环境

进入对应方向的文件夹，按其 `README.md` 中的要求进行开发。**强烈建议**将完整代码托管在一个**独立的公开仓库**中，主仓库这边只放最终报告与必要的元信息文件——这样既方便你单独展示项目，也避免主仓库被大量数据/权重污染。

### Step 3：报告命名

最终报告须严格遵循以下命名格式：

```
学号-姓名-方向编号-方向简称.md     例：20231234-张三-02-Jailbreak.md
学号-姓名-方向编号-方向简称.pdf    例：20231234-李四-03-KnowledgeEditing.pdf
```

附录文件可在主名后追加描述性后缀，如 `20231234-张三-02-Jailbreak-评测日志.ipynb`。

### Step 4：提交 Pull Request

- **PR 标题**：`[最终作业] 学号 姓名 - 方向简称 - 项目标题` 示例：`[最终作业] 20231234 张三 - Jailbreak - 基于梯度引导的可迁移越狱攻击与 SmoothLLM 防御对比`
- PR 描述至少包含
  - ✅ 独立代码仓库链接
  - ✅ Demo / 视频链接（如有）
  - ✅ 主要成果与创新点（≤ 200 字）
  - ✅ 主要评测指标的对比表
  - ✅ 自评亮点与已知不足

## ⚖️ 安全与学术伦理

- 涉及越狱攻击或对抗性研究的实验，**仅限于开源模型或经课程授权的 API**，严禁针对未授权的商用线上系统发起真实攻击。
- 报告与代码中**禁止包含可直接复现的危险指引全文**；高危类别样本请以编号或截断形式呈现。
- 所有代码与报告须由本人独立完成；如组队请在 Issues 中申请并明确分工。
- 引用他人代码、Prompt、数据集时，请在 README 与报告中明确标注来源与许可证。
- 鼓励使用免费算力（Colab / Kaggle / AutoDL / 阿里天池等）。
